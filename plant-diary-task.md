# Дневник растений

## Цель
Собрать Android-приложение на Kotlin для учета растений и ухода за ними в соответствии с заданием пользователя и требованиями курса `android-kotlin-course`.

## Источники
- Пользовательское задание из чата.
- Требования курса: https://github.com/ivanshchitov/android-kotlin-course/blob/master/requirements.md
- Репозиторий курса: https://github.com/ivanshchitov/android-kotlin-course
- Репозиторий проекта: https://github.com/Asagiry/PlantDiary
- Дата сверки ссылок: 2026-03-11

## Подтвержденные параметры
- Имя приложения: `Plant Diary`
- Package / applicationId: `com.asagiry.plantdiary`
- Git remote: `https://github.com/Asagiry/PlantDiary.git`
- По структуре данных `CareRecord` трактуется как отдельная сущность, потому что по заданию для записей ухода нужен самостоятельный CRUD и списки по конкретной дате.

## Функциональные требования из задания

### Сущность растения
Каждая запись о растении должна содержать:
- Наименование.
- Описание.
- Тип: `комнатное` или `садовое`.
- Интервал полива в днях.
- Дату и время высаживания, если растение садовое.

### Сущность ухода за растением
Каждая запись об уходе должна содержать:
- Растение: наименование и тип.
- Дату следующего полива.
- Специальную пометку, если полив нужен сегодня.
- Дату высаживания, если растение садовое.
- Специальную пометку, если высаживание нужно сегодня.

### Обязательные сценарии
- Просмотр списка растений.
- Добавление растения.
- Редактирование растения.
- Удаление растения.
- Просмотр записей об уходе.
- Добавление записи об уходе.
- Редактирование записи об уходе.
- Удаление записи об уходе.
- Просмотр списка растений для полива на:
  - сегодня;
  - завтра;
  - выбранный пользователем день.
- Просмотр списка растений для высаживания на:
  - сегодня;
  - завтра;
  - выбранный пользователем день.
- Фильтрация растений по типу.
- Поиск растений по наименованию.

## Обязательные технические требования курса
По `requirements.md` приложение должно соответствовать следующим ограничениям и ожиданиям:

- Приложение должно быть написано на Kotlin под Android.
- Навигация должна быть собрана через Navigation Component с nav graph.
- Нужно использовать MVVM.
- Нужно использовать Room для постоянного хранения данных.
- Нужно использовать Data Binding.
- Необходимо корректно сохранять состояние при повороте экрана.
- Интерфейс должен поддерживать локализацию.
- Пользователь должен иметь возможность просматривать, добавлять, редактировать и удалять данные.
- Приложение должно показывать помощь по использованию.
- Нужно обрабатывать некорректный ввод и не падать на базовых пользовательских сценариях.

## Предварительная структура реализации
Ниже не финальное решение, а каркас, который безопасно начать после подтверждения блокирующих деталей:

- `Plant`:
  - `id`
  - `name`
  - `description`
  - `type`
  - `wateringIntervalDays`
  - `plantingDate`
  - `plantingTime`
- `CareRecord`:
  - `id`
  - `plantId`
  - `nextWateringDate`
  - `plannedPlantingDate`
  - `plannedPlantingTime`
